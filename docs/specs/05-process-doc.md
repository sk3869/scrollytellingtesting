# Spec 05 — PROCESS.md

## Purpose
Document how the project was built with AI assistance, satisfying the assignment's explicit grading requirement.

## What done looks like
- `PROCESS.md` exists at the repository root.
- It contains exactly three sections as required by the assignment brief:
  1. **One feature, the prompts you used** — the actual prompts (or faithful summary) sent to the AI for one chosen feature, in order, unedited.
  2. **Exit checks — pass and fail** — the exit checks defined for that feature, each marked pass or fail.
  3. **Failure-mode reflection** — one paragraph identifying which of the four failure modes (short memory, invents when unsure, does more than asked, cannot tell you it is lost) was encountered, with a concrete example and what was changed.
- Length is approximately one page.
- Tone is honest — mistakes are included, not smoothed over.

## Constraints
- Must reference a real feature from this project (not a hypothetical).
- Exit checks must be concrete and runnable (e.g. "npm run test passes" not "it works").
- The failure-mode section must name one of the four modes from the brief exactly.

## Non-goals
- Covering every feature built — one is enough.
- Listing all prompts from the entire project.
